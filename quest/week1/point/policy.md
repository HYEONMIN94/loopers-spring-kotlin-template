# 포인트 도메인 정책

## 요구 데이터

- 포인트 충전을 위해 다음 정보를 입력해야 합니다:
    - **X-USER-ID**: 요청자의 식별자 (헤더를 통해 전달)
    - **충전 금액**: 정수, 1 이상

---

## 도메인 정책

- 포인트는 **사전 충전 방식**으로 결제되며, \*\*실제 결제(PG)\*\*는 구현 대상이 아닙니다.
- 포인트 **사용은 주문 과정에서 자동 처리**되며, 별도 사용 API는 제공되지 않습니다.
- 포인트 충전 시 다음 조건을 맞춰야 합니다:
    - **0 이하의 금액 충전 시 예외 발생**
    - **존재하지 않는 유저 ID로 충전 요청 시 예외 발생**
- 포인트 충전이 성공하면, **충전된 보유 채용**을 응답으로 발행합니다.
- 존재하지 않는 유저로 요청할 경우, 404 Not Found 응답을 반환합니다.

---

## 조회 관련 정책

- 포인트 조회 시 다음 조건을 맞춰야 합니다:
    - **요청 헤더에 X-USER-ID 가 존재해야 합니다**
        - 없을 경우, **400 Bad Request 응답 발행**
    - 해당 ID의 회원이 존재하면, **보유 포인트 발행**
    - 존재하지 않는 회원인 경우:
        - **null 발행** 또는
        - **404 Not Found 응답 발행**

---

## 시스템 정책

- 모든 포인트 관련 요청은 **`X-USER-ID` 헤더를 통해 유저를 식별**합니다.
